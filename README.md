# タスク管理アプリ

## アプリ概要
タスクを登録・管理するためのWebアプリケーションです。

タスクごとにタイトル・説明・締切日時・完了状態を設定でき、登録後は一覧・詳細・編集・削除を行うことができます。

---

## 使用技術

- .NET 8
- ASP.NET Core Razor Pages
- SQLite
- Entity Framework Core
- Bootstrap

---

## 実装機能

### 基本機能（CRUD）
- タスク作成
- タスク一覧表示
- タスク詳細表示
- タスク編集
- タスク削除

### 追加機能
- タスク完了／未完了切替
- 説明（メモ）入力
- 締切日時設定
- 作成日時表示
- 完了済みタスク削除時の確認ダイアログ

## AIの活用方法

開発にはChatGPTを活用しました。

私はこれまで主にPythonの学習経験があり、C#およびASP.NET Core Razor Pages、Entity Framework Coreは未経験でした。

そのため、以下の場面でAIを活用しました。

- 開発環境のセットアップ（.NET SDK、VS Code、NuGetパッケージ導入など）
- Razor Pagesの画面（.cshtml）と処理（.cshtml.cs）の役割理解
- Entity Framework Coreを利用したCRUD実装方法の確認
- SQLiteとの接続設定、Migration実行方法の確認
- エラー発生時の原因整理と解決方針の検討
- UI改善や機能追加時の実装相談

AIが提案した内容をそのまま適用するのではなく、
コード内容を確認しながら修正・実行・検証を繰り返して完成させました。

特に、環境構築から動作確認まで一連のセットアップ（キッティング）を自身で行い、対話しながら試行錯誤して実装を進めました。

また、要件を満たしたうえで追加機能（締切日時、完了状態切替、削除確認など）を実装し、対話形式でアイデアを発展させながら開発を進めました。

- 基本要件に加えて締切日時機能を追加しました。

- 完了済みタスクを削除する際に確認ダイアログを表示し、誤操作を防ぐようにしました。

---

## 苦労した点と解決方法

開発中にEntity Framework Core関連の参照エラーや、DbContextの依存関係エラーが発生しました。
また、C#およびASP.NET Coreは未経験だったため、Entity Framework Coreの導入やデータベース接続部分に苦戦しました。

解決に向けて、
- 必要なNuGetパッケージを追加
- DbContext登録を見直し
- Migrationを再実行

することで正常に動作する状態に改善しました。

また、Razor Pagesの画面（.cshtml）と処理（.cshtml.cs）の役割分担を意識しながら実装しました。

```

```

へアクセスしてください。
